# 입력 포맷 → HWPX 변환 품질 감사

기준일: 2026-06-24

## 평가 방법

점수는 원본 화면의 픽셀 복제율이 아니라 서비스가 약속한 범위 안에서의 `내용 보존 35% + 구조 보존 30% + HWPX 호환 구조 20% + 실패 안전성 15%`로 산정했다. Chromium에서 실제 업로드·변환·다운로드한 HWPX를 다시 열어 XML/ZIP을 검사했으며, 한컴의 최종 시각 렌더링만 수동 확인 영역으로 남긴다.

## 이번 감사에서 수정한 결함

1. 텍스트 입력을 항상 UTF-8로 읽어 TXT의 EUC-KR 지원 안내와 실제 동작이 불일치하던 문제.
2. IPYNB 코드 셀을 코드 블록이 아닌 일반 문단으로 출력하던 문제.
3. 손상 JSON/IPYNB가 실패하지 않고 오류 안내문 HWPX를 만들던 문제.
4. HWP5·알 수 없는 HWP·본문 없는 HWPX가 내용 대신 안내문 HWPX를 만들던 문제.
5. SheetJS 미로드·빈 XLSX가 정상 결과처럼 처리될 수 있던 문제.
6. 닫히지 않은 CSV 따옴표가 조용히 표로 변환되던 문제.
7. JSON 객체 배열이 행형 표가 아니라 JSON 문자열/다수 표로 펼쳐지던 문제.
8. IR 직접 입력의 표 셀·runs·인용구 내부 XML 금지 제어문자 정규화 누락.
9. HTML에서 들여쓴 `<li>`와 중첩 목록이 사라질 수 있던 문제.
10. HTML `rowspan`/`colspan`이 HWPX 병합 셀로 전달되지 않던 문제.
11. 용지 방향을 enum 또는 폭·높이 한쪽만으로 판단해 두 차례 회귀가 발생했다. v4.5.7에서 회전 전 기본 치수 유지 + 세로 `WIDELY`/가로 `NARROWLY` + 회전 후 유효 본문 폭 계산으로 정정하고 8조합 행렬 테스트를 추가했다.
12. 패키지 게이트가 header/section 외 XML well-formed를 확인하지 않던 공백.
13. DOCX 그림의 `hc:img → content.hpf → BinData → manifest` 연결을 검사하지 않던 공백.

## 검증 명령과 판정

- `npm run test:golden`: 정상 11개(MD, HTML, TXT UTF-8/EUC-KR, CSV, XLSX, JSON 일반/IR, IPYNB, DOCX) + Lab/UX + 실패 입력 5개.
- `node qa/gate.js qa/fixtures/md_hwpx_test.md`: 패키지 7영역.
- `node qa/gate.js qa/fixtures/docx_table_test.docx`: 병합·중첩 표 격자.
- `node qa/gate.js qa/fixtures/docx_image_test.docx`: 그림 참조 4단 연결.
- `node qa/gate.js tests/fixtures/sample.xlsx`: XLSX 표 패키지.

자동 PASS는 “파일 구조가 유효하다”는 뜻이다. 실제 한컴 화면에서 가로 용지, DOCX 그림, 병합 표, 코드 배경·글꼴을 열어 보는 것이 최종 승인 조건이다.
