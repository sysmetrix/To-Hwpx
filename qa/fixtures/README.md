# QA Fixtures

브라우저 로컬 변환 회귀 테스트용 작은 입력 파일입니다.

## 종합 회귀 입력 + 자동 게이트

- `md_hwpx_test.md` — H1~H6·중첩목록·태스크리스트·표·코드블록·취소선·밑줄·HTML인라인·HR 종합 입력
- `sample.docx` — 제목(영문/한글/outlineLvl) + 굵게/기울임/밑줄/취소선/글자색 DOCX 입력
- `docx_table_test.docx` — 가로/세로 병합 + **중첩 표** + 일반 표 DOCX 입력 (표 격자 무결성 회귀 방지)
- `../samples/*.hwpx` — 위 입력의 변환 통과본(참고용, 한컴오피스로 직접 열어 시각 확인)
- 자동 검증: 저장소 루트에서 `node qa/gate.js qa/fixtures/md_hwpx_test.md` (①~⑥ 모두 PASS면 exit 0). 사용법은 `qa/gate.js` 상단 주석 참고.
  - ⑥ 표 격자 무결성: 모든 표의 (행,열)이 span 반영 시 정확히 1회 덮이는지 검사 — 중첩 표 누수·들쭉날쭉한 행이 만드는 "한컴이 안 열리는 깨진 표" 차단

## 포맷별 수동 케이스

텍스트 계열은 이 폴더의 파일을 직접 업로드합니다.
DOCX/XLSX/HWPX는 실제 오피스 문서 구조가 필요하므로 아래 수동 케이스를 함께 사용합니다.

- DOCX: 제목, 본문, 표, 굵게/기울임, 이미지 1개가 있는 문서를 만들고 이미지 손실 안내가 표시되는지 확인
- XLSX: 첫 번째 시트에 표, 숫자, 긴 한글 셀, 빈 셀을 포함
- HWPX: 이 앱에서 생성한 HWPX를 다시 업로드해 텍스트 재추출 가능 여부 확인
- HWP: HWP5 바이너리 파일을 업로드해 변환 안내 메시지 확인
- 손상 ZIP: 임의 텍스트 파일의 확장자를 `.docx`로 바꾸어 파싱 실패가 HWPX 생성으로 이어지지 않는지 확인
- 대용량: 텍스트 100MB 초과, 바이너리 50MB 초과 파일로 사전 차단 메시지 확인
