바이브 코딩 활용법

1. 빠른 데모

브라우저에서 바로 보여줘야 하면 rhwp나 TotalDocs 계열처럼 프런트엔드 연동이 쉬운 저장소를 우선 고르세요. “업로드 즉시 미리보기” 같은 PoC에 가장 빠릅니다.

2. RAG 수집

문서 내용을 Markdown이나 정규 텍스트로 바꿔야 하면 hwp2md, HWPCONV, deepsearch 같은 파서형 저장소가 효율적입니다. 공공기관 공지, 회의자료, 사업계획서 수집기에 잘 맞습니다.

3. 엔진 내재화

장기적으로 기관 서비스에 넣으려면 Rust 계열 저장소가 유리합니다. 성능, 배포 안정성, 구조 통제력이 좋아서 서버·웹·데스크톱 확장성이 큽니다.

근거 출처는 각 저장소 카드와 본문에 표시했습니다. 핵심 공개 근거는 rhwp 저장소 README와 GitHub 코드 검색 결과입니다. [page:1][cite:2]
