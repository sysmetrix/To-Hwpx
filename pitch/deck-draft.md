## 2. 문제

공공기관·학교·행정 현장은 여전히 한글(HWP/HWPX) 문서 제출을 요구합니다. 하지만:

- **한컴오피스가 없거나 라이선스가 없는 사람**이 많습니다 — 개인 PC, 임시 근무자, 외부 협력사
- 이미 만든 문서(Markdown 회의록, Word 보고서, 엑셀 표, Jupyter 분석 노트북, PPT 발표자료)를 **매번 손으로 다시 한글 문서로 옮겨 적는** 비효율이 반복됩니다
- 온라인 변환기는 있지만, **파일을 서버에 업로드**해야 해서 공공기관·개인정보가 담긴 문서에는 쓰기 부담스럽습니다

**한 줄 요약**: "한글 문서로 내야 하는데, 한글도 없고, 업로드는 불안하다."

---

## 3. 솔루션

**To HWPX**는 브라우저 안에서 끝나는 문서 변환기입니다.

- MD·DOCX·PPTX·HTML·CSV/XLSX·JSON·TXT·IPYNB·HWP → **HWPX**로 즉시 변환
- 설치 프로그램 없음, 회원가입 없음
- 파일은 **당신의 브라우저를 떠나지 않습니다** — 변환이 전부 클라이언트에서 일어남(서버 업로드 없음)
- 무엇이 보존되고 무엇이 빠지는지 **포맷마다 미리 안내** — "완벽 변환"이라고 과장하지 않음

---

## 4. 어떻게 동작하는가

```
파일 선택 → 공통 중간 표현(IR)으로 정규화 → HWPX(ZIP+XML) 생성·검증 → 다운로드
```

- 입력 포맷이 무엇이든 같은 내부 표현(IR)으로 바뀌므로, 새 포맷을 추가해도 출력 로직은 그대로 재사용됩니다
- 생성된 HWPX는 자체 구조 검증(mimetype, XML well-formed, ID 참조 무결성, 표 격자, 그림 참조, 링크 필드)을 통과한 것만 다운로드됩니다

## 8. 기술 스택 & 신뢰 설계

- 순수 브라우저 앱(빌드 과정 없음) — JSZip으로 HWPX(ZIP+XML) 직접 생성
- 콘텐츠 보안 정책(CSP)으로 스크립트 실행 경로 제한, XSS 방어
- 유일한 외부 네트워크 요청은 CDN 라이브러리 로드와, 사용자가 Markdown에 명시한 원격 이미지뿐
- 손상 파일·용량 초과·Zip Bomb 방어 등 실패를 조용히 감추지 않고 명확히 안내
